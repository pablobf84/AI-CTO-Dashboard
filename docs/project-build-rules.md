# AI CTO Dashboard — Build Rules

## Reglas absolutas
1. No generar demos ni placeholders.
2. No dejar TODOs ni FIXME.
3. Todo debe quedar listo para compilar.
4. Multi-tenant real por organización.
5. TypeScript estricto.
6. Validación en cliente y servidor.
7. Manejo de errores visible.
8. RLS correcto en Supabase.
9. Si se modifica un archivo, devolverlo completo.
10. Antes de construir, resumir plan técnico.
11. Construir por fases cerradas.
12. Nada de mock data persistente.

## Criterio de calidad
- seguridad antes que estética
- estructura antes que automatización avanzada
- flujo principal vivo antes que extras

## Criterio de terminado del Sprint 1
No está terminado si no funcionan:
- registro/login
- creación de organización
- creación de proyecto
- formulario de intake
- guardado real en base de datos
- navegación protegida
